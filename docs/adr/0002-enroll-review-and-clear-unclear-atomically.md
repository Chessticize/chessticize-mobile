# Enroll in Review and clear the initiating Unclear marker atomically

When Manual Review Enrollment is initiated from an Unclear Attempt, Chessticize creates the Review Schedule and clears that attempt's Unclear marker as one atomic domain/store operation. Both changes commit or neither does, while enrollment initiated without an attempt creates only the schedule. Synchronization preserves the same invariant: the enrollment clear defeats older marker state, while a genuinely later explicit re-mark may make the attempt Unclear again. The UI reports success only after the atomic commit; a failed enrollment retains the unscheduled state and Unclear marker and offers retry. This prevents relaunch, persistence, synchronization, or optimistic UI from exposing a half-completed user action and preserves the attempt-scoped marker boundary.
