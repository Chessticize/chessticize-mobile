# Publish one Android build through Play and GitHub

Android will have one production identity, `com.chessticize.mobile`, distributed through Google Play and GitHub Releases rather than separate store and sideload editions. Each release uploads the validated Android App Bundle to Play App Signing, promotes Production directly to 100 percent after Internal and Closed testing, obtains the universal `arm64-v8a` plus `x86_64` APK signed by the production app-signing key, and publishes that APK with its SHA-256 checksum in the matching GitHub Release. A manually triggered workflow uses a protected environment and a least-privilege Play service account to prepare a draft release, verify package, version, signing certificate, ABIs, and size, and waits for human approval before publication. GitHub APK users update manually through an explicit Releases link in Settings; the app does not poll, download, or install updates in the background. The public version name is shared with iOS while Android `versionCode` and iOS `CURRENT_PROJECT_VERSION` advance independently, and Android uses platform-specific tags such as `android-v1.1.0-build-1`. Separate package names, signing identities, independently built GitHub APKs, background update channels, generic cross-platform version tags, and staged Production percentages are rejected so installs can update across channels and release evidence refers to one build.
